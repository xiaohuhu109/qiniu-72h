# 幕笺 - 小说转剧本工具

## 项目概述

幕笺是一款在微信原生小程序环境下运行的小说转剧本工具。支持作者将长篇小说文本（粘贴或 TXT 上传）一键转换为结构化的剧本，提供书架式管理、按场次查看、单场编辑与导出功能，全部数据保存在小程序本地存储。

## 五、核心功能与规则

### 5.1 剧本生成规则

| 版本 | 包含字段 |
|------|----------|
| 完整版 | 场次、地点、内外景、时间、角色、动作、台词 |
| 精简版 | 场次、地点、内外景、时间、角色、台词（省略动作） |

### 5.2 场景拆分规则

- 以"换地点"或"换时间"或"情节转折"为准划分新场次
- 每个场次是一个完整的叙事单元
- 场次数目控制在 3~30 之间

### 5.3 存储规则

- 所有数据保存在小程序 `localStorage`
- Key: `novel2script_books`
- 每本小说独立存储，重启不丢失

## 六、AI 分析模块

### 6.1 配置信息

```javascript
const CONFIG = {
  url: 'https://api.deepseek.com/v1/chat/completions',
  model: 'deepseek-chat',
  timeout: 60000
};
```

### 6.2 功能说明

- 调用 DeepSeek API 进行智能场景分析
- 自动识别：场景切分、角色提取、对话提取、地点/时间标注
- 支持文本截断（最大 60000 字符）

### 6.3 输出格式

```json
{
  "title": "小说标题",
  "scenes": [
    {
      "location": "场景地点",
      "env_type": "内景/外景",
      "time": "时间",
      "primary_character": "主要角色",
      "characters": ["角色1", "角色2"],
      "action": "关键动作摘要",
      "dialogues": [
        {"character": "角色名", "text": "对话内容"}
      ]
    }
  ]
}
```

## 九、技术栈说明

| 技术 | 说明 |
|------|------|
| 框架 | 微信小程序原生 |
| 存储 | wx.setStorageSync / wx.getStorageSync |
| AI 接口 | DeepSeek API |
| 文件操作 | wx.getFileSystemManager |

**选择原生开发的理由：**
- 包体积更小，加载更快
- 微信开发者工具直接调试，效率更高
- 单平台场景下无需跨端框架

---

## 十、开发注意事项

1. **无后端**：所有处理在小程序端完成
2. **文本长度**：过长文本需截断处理（建议最大 60000 字符）
3. **API Key**：使用 AI 功能需在 `utils/ai-analyzer.js` 中配置 DeepSeek API Key
4. **导出限制**：小程序文件系统受限，使用兼容 API 导出
